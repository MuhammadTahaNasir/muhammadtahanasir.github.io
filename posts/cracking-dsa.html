C++ and Data Structures at Uni

At uni, we’re grinding through data structures in C++, and it’s like switching from a bike to a spaceship. Our professor (big props to Zia Sir from my Python days) has us coding arrays, linked lists, and stacks from the ground up. C++ is intense pointers and memory management are no joke, but it’s making me see how data structures actually work. Check out this C++ linked list node I coded for class:


#include 
struct Node {
    int data;
    Node* next;
    Node(int val) : data(val), next(nullptr) {}
};
class LinkedList {
public:
    Node* head;
    LinkedList() : head(nullptr) {}
    void insert(int val) {
        Node* newNode = new Node(val);
        newNode->next = head;
        head = newNode;
    }
};

This code was my “aha” moment, linked lists are like a chain of treasure chests, each pointing to the next. C++’s raw power helped me get why DSA is the backbone of efficient ML code.

Not as Tough as They Say

But after uni classes and some LeetCode practice, I’m like, “Bro, it’s not *that* bad!” Maybe I’m a bit cocky, but arrays, linked lists, and stacks are starting to feel like squad mates. The trick? Why Data Structures Matter

Data structures are the backbone of efficient programming, especially in fields like machine learning and AI. They help organize and manage data, making code faster and more scalable. Mastering arrays, linked lists, trees, and stacks builds a solid foundation for solving real-world problems. With resources like Striver's DSA sheet, Abdul Bari’s explanations, and platforms like LeetCode and GeeksforGeeks, anyone can level up their problem-solving skills and become a better developer.
