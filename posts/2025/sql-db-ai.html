My Database Journey Begins

Yo, back in the day, I’d dodge anything with “data” in it like it was a bouncer in cricket. But this semester at FAST NUCES under Sir Shoaib’s guidance,flipped the script. My database course threw me into MySQL, SQLite, Neo4j, and even discovered vector databases, which are straight-up crucial for AI. From building schemas to querying graphs, I learned how data powers everything.

MySQL and SQLite: The Relational Roots

MySQL and SQLite were my first stop. MySQL is like the captain of relational databases structured, reliable, perfect for business apps like e-commerce. SQLite, its lightweight cousin, is the go-to for mobile apps or IoT, no server needed. I learned to design tables, normalize data (1NF, 2NF, 3NF), and write SQL queries with joins and aggregates. Think of it like organizing your cricket team’s stats runs, wickets, averages all in neat tables. Here’s a sample SQL query I wrote:


SELECT p.player_name, AVG(s.runs) as avg_runs
FROM players p
JOIN scores s ON p.player_id = s.player_id
GROUP BY p.player_name
HAVING AVG(s.runs) > 50;

This query finds players averaging over 50 runs clean and efficient, just like a well-placed cover drive.

Neo4j: Grappling with Graphs

Then came Neo4j, the graph database king. Unlike tables, it’s all about nodes (e.g., people, movies) and relationships (e.g., friends, acted in) think your social media network, where friends are connected. We can say it's like mapping your WhatsApp group’s connections. It’s all about nodes and relationships. I learned Cypher, Neo4j’s query language, to model stuff like recommendation systems. It’s perfect for AI apps where relationships matter, like suggesting friends or products. Here’s a Cypher query to find friends of friends:


MATCH (user:Person {name: "Taha"})-[:FRIEND]->(friend)-[:FRIEND]->(fof)
RETURN fof.name

This query is like asking, “Who’s in my extended squad?” Neo4j made me see how AI can leverage connections for smarter predictions.

Vector Databases: The AI Game-Changer

Here’s where it got wild: vector databases. These bad boys store high-dimensional vectors, think numerical representations of words, images, or user preferences. They’re the backbone of AI apps like ChatGPT or image search. Instead of exact matches, they find *similar* items using approximate nearest neighbor (ANN) search. Picture searching “biryani” and getting recipes with similar vibes, not just exact matches. Vector DBs like Pinecone or Weaviate power semantic search and Retrieval Augmented Generation (RAG) in LLMs. They’re why AI feels so smart.

Assignments That Leveled Me Up

My two assignments were like batting practice against spinners means tough but rewarding. Check them out on my GitHub!

Assignment 2: Expense Tracker (SQLite)
I built a desktop app using Python, Tkinter, SQLite, and Matplotlib. It lets users track expenses and income, with a dark-themed GUI, user authentication, and charts (pie for expense breakdown, bar for income vs. expense). The SQLite database had tables for users, expenses, and income, with foreign keys for integrity. I learned to design schemas and query data, like tracking my biryani budget!

Assignment 3: Movie Graph (Neo4j)
I created a movie graph using Neo4j, loading data from movies.csv. I wrote Cypher queries to create nodes (Movie, Director, Actor, Genre, Country) and relationships (DIRECTED_BY, ACTED_IN, IN_GENRE, PRODUCED_IN). For example, I linked movies to genres and actors to films, perfect for a Netflix-style recommendation system. This showed me how graphs shine in AI for finding patterns.

Why Databases Matter for AI

Databases are the backbone of AI. MySQL and SQLite handle structured data like user profiles in my upcoming Database Semester's Project , Dishly - (Smart Resources Management System). Neo4j powers recommendation systems by mapping relationships. Vector databases like Pinecone or Weaviate make AI smart by finding patterns in embeddings, semantic search or RAG in LLMs. This semester showed me databases aren’t just storage; they’re the engine for real-world AI.
